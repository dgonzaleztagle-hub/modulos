# Catálogo

Inventario consolidado por repo.

## Repos con fichas iniciales

- `acargoo`: delivery de carga, pricing, state machine, tracking, POD, superficies multi-rol
- `agente`: bridge remoto por Telegram para operar una interfaz local y devolver respuesta por archivo
- `agendaproyectos`: workspace operativo por proyecto, accounting tab, vault liviano y normalización de workspace
- `apk germain`: artefactos Android/TWA de distribución para Germain, útiles como referencia de publicación móvil pero no como fuente de lógica
- `apimiel`: referencia visual de catálogo inmersivo premium con ficha técnica modal y dirección de arte fuerte
- `bizscout`: lead hunting, clasificación comercial, wizard de outreach y agenda de seguimiento
- `caprex`: biblioteca privada de contenido con grants temporales, cuentas compartidas y ADN heredado desde `zeus`
- `Centro de padres`: referencia de vertical escolar/comunitaria todavía sin familia modular clara
- `fidelizacion`: booking por rubro, agenda con push y tenancy para negocios de servicios
- `dondedario`: referencia gastronómica estática con valor de branding, sin motor nuevo
- `rishtedar`: reservas, delivery, loyalty, scanner, dashboard operativo, pagos
- `truckpos_new`: caja, importación de productos, tickets, multi-sucursal, reportes, pagos
- `zeus`: Zeleri, booking locks, agenda de servicios, acceso digital
- `hojacero`: agenda, sales-agent, territorial, radar, pipeline, inbox, vault, CMS, food-engine, UI premium, scrapers, factory, B2B store
- `ICEBUIN`: tooling de catálogo, parser Excel, mapeo de imágenes y scripts de contenido
- `montemadero`: prototipo visual con IA Studio, sin modularidad fuerte detectable
- `pluscontable.cl`: tenancy, onboarding, tenant shell, PDFs RRHH/honorarios, pagos, AI chat, mensajería
- `pluscontableapisii`: integración tributaria SII bajo demanda con scraping backend y sync paralelo
- `kurso/remix-of-pagos-pre-kinder-b`: billing SaaS, payment grouping, branding tenant, onboarding público, soporte, formularios, onboarding kit
- `CorredoresPro`: multi-tenant inmobiliario, publisher externo, contratos/PDF, portales, mensajería, onboarding
- `reparpads`: referencia visual secundaria con patrón menor de activación remota
- `superpanel3.0`: visit tracker, integración con AgendaProyectos, snapshot/backup, rate limit, costo ponderado y billing operativo
- `test-remotion`: laboratorio pequeño de video/render con Remotion, útil como semilla audiovisual
- `ceroriesgos`: IA vertical CPHS, generación de PDFs operativos y patrón de IA incrustada en workflow

## Observación de esta fase

- `truckPos` no tiene todavía ficha profunda porque en esta pasada solo apareció `_management_data`
- `metaads` apareció como repositorio principalmente documental/estratégico, no como fuente fuerte de módulos de código
- `ROI` se comporta como landing/calculadora muy pequeña, sin cantera modular relevante
- `Municipalidad` no mostró código explotable en esta pasada, solo material de instrucciones
- `cinematic-explore`, `copgame`, `juego` e `ia sin humos el libro interactivo` se ven más como laboratorios creativos o prototipos de contenido que como fuentes del stack modular principal
- `backup`, `cloudlab.cl` y `claude-code-templates` se comportan como carpetas auxiliares, empaquetados o material de apoyo, no como repos fuente para extracción modular
- el catálogo actual prioriza utilidad comparativa por encima de cobertura exhaustiva del código
